1 Module Overview

This module implements the complete ride pool lifecycle — from a driver publishing a ride to a rider discovering, requesting, and getting approved for that ride.

Two User Roles

RoleActions
DriverCreate ride pool → Review join requests → Approve / Reject riders
RiderSearch rides → Filter by preferences → Send join request → Track request status

Driver Creates a Ride Pool With:

  • ✅ Pickup & Drop Locations — Exact or approximate locations with GPS coordinates
  • ✅ Departure Time & Date — When the ride will begin
  • ✅ Available Seats — Maximum passengers allowed (AI-detected from car photo)
  • ✅ Vehicle Details — Car model, license plate, verified photo
  • ✅ Preferences & Rules — Music, smoking, pet-friendly, female-only, luggage size

Rider Joins a Ride Pool By:

  • ✅ Searching for available rides based on destination and departure time
  • ✅ Filtering rides based on preferences (female-only, no smoking, etc.)
  • ✅ Requesting to join a ride and waiting for driver approval
  • ✅ Once approved, receiving ride details including pickup location

Key Design Decisions

✅ UUID for ride IDs — Prevents sequential ID enumeration attacks. Users can't guess ride URLs.
✅ Separate total_seats vs available_seats — available_seats decrements on approval; total_seats stays fixed.
✅ Coordinates as text — Avoids floating-point precision issues. Parsed to float in application code.
✅ Route polyline in DB — Avoids re-calling OSRM for every page view. ~500 bytes for a 100km route.

7.2 — GET /api/rides (Search Rides)

What it does: Searches rides with location proximity + date + seat filters.

[All Active Rides]
  ↓ SQL: status = 'active'
  ↓ SQL: departure_date = exact date (timezone-safe)
  ↓ SQL: available_seats >= requested
  ↓ JS: Haversine(rider_pickup, ride_pickup) ≤ 30 km
  ↓ JS: Haversine(rider_drop, ride_drop) ≤ 30 km
  → [Filtered Results]

Haversine Formula

Calculates great-circle distance between two GPS points on Earth's surface:

function haversine(lat1, lng1, lat2, lng2) {
  const R = 6371; // Earth radius in km
  const dLat = ((lat2 - lat1) * Math.PI) / 180;
  const dLng = ((lng2 - lng1) * Math.PI) / 180;
  const a = sin(dLat/2)² + cos(lat1) * cos(lat2) * sin(dLng/2)²;
  return R * 2 * atan2(√a, √(1-a)); // Distance in km
}

7.3 — POST /api/rides/[id]/request (Join Request)

5-step validation:
1. ✅ User is authenticated (Clerk JWT)
2. ✅ Ride exists in database
3. ✅ User is NOT the driver (can't join own ride)
4. ✅ Enough seats available
5. ✅ No duplicate pending request → Insert with status "pending"

7.4 — PATCH /api/rides/[id]/request (Approve/Reject)

On Approve ✅

  1. Verify user IS the driver
  2. Check seats still available
  3. Update status → "approved"
  4. Decrement available_seats

On Reject ❌

  1. Verify user IS the driver
  2. Update status → "rejected"
  3. Seats remain unchanged

10 Security & Validation

ConcernHow We Handle It
AuthenticationClerk middleware protects all write APIs. auth() extracts userId from JWT.
AuthorizationOnly ride's driver can approve/reject. Checked via ride.driverId === user.id.
Self-join preventionAPI rejects requests where riderId equals driverId.
Duplicate preventionChecks for existing "pending" request. Returns 409 Conflict.
Seat overflowBefore approving, re-checks available_seats >= requested_seats.
UUID ride IDsPrevents URL enumeration. IDs like ed785c99-bb92-4df2-...
Image validationGroq AI verifies uploaded image is actually a car before storing.
Address normalizationcleanAddress() replaces Arabic commas (،) with standard commas.
Timezone safetyDates parsed with T00:00:00 suffix to prevent UTC midnight shift in IST.

11 Key Algorithms

🌍 Proximity Search (Haversine)

Rider searches "GLA University → Agra"

  1. Geocode both locations → GPS coords
  2. Fetch all active rides from DB
  3. For each ride: calculate distance between search point and ride point
  4. Include only if both pickup AND drop within 30 km

⏰ Ride Expiration

Auto-disable expired rides on dashboard:

departure_time + duration < now
→ Show "Expired" badge
→ Reduce opacity
→ Disable join button
